# Figure Test

This document tests how image embeds are registered as numbered figures and how wikilinks are resolved to them.

# Case 1 — Embed with Caption Becomes Numbered Figure

Expected behavior: the image appears as a framed figure with caption below. The header "Figure 1: …" is automatically generated by LaTeX with Babel-English.

![[neuron.excalidraw.png|Diagram of a Neuron]]

As seen in [[neuron.excalidraw.png]], the model has the typical structure.

The reference can also have its own text; see [[neuron.excalidraw.png|here]].

# Case 2 — Embed Without Caption Uses the Filename

Expected behavior: the image is also rendered as a numbered figure; since no caption is specified, the filename serves as the caption. The anchor is registered on the source path.

![[neuronales_netz.excalidraw.png]]

Here is a reference: [[neuronales_netz.excalidraw.png]].

# Convention: Wikilink Fallback

Wikilinks to non-embedded images are — like note wikilinks — passed through as plain text (this is intentionally not a crash, so that references to non-embedded targets remain possible as pure conceptual references).

- `[[neuron.excalidraw]]` (without `.png`) → plain-text fallback, because the map key strictly requires the full spelling including extension.
- `[[does-not-exist.png]]` → also plain-text fallback.
